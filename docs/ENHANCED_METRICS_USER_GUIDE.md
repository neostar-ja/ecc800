## 2. การนำทางสู่หน้าเมตริก

### เมนูการนำทาง

1. **คลิกที่เมนู "📊 Metrics"** ในแถบเมนูด้านซ้าย

### หน้าเมตริกขั้นสูง

- **แผงควบคุม:** สำหรับเลือกไซต์, อุปกรณ์, และตั้งค่าต่างๆ
- **สรุปอุปกรณ์:** แสดงภาพรวมเมตริกสำคัญ
- **เมตริกตามหมวดหมู่:** จัดกลุ่มเมตริกเป็นหมวดหมู่

## 3. การใช้งานแผงควบคุม

แผงควบคุมอยู่ด้านบนของหน้า ประกอบด้วย:

### 3.1 เลือกไซต์ 🏢

- **DC (Data Center):** ศูนย์ข้อมูลหลัก
- **DR (Disaster Recovery):** ศูนย์ข้อมูลสำรอง

**วิธีใช้:**
1. คลิกที่ dropdown "🏢 เลือกไซต์"
2. เลือกไซต์ที่ต้องการ (DC หรือ DR)
3. ระบบจะโหลดข้อมูลอุปกรณ์ของไซต์นั้น

### 3.2 เลือกอุปกรณ์ 🔧

หลังจากเลือกไซต์แล้ว:

1. คลิกที่ dropdown "🔧 เลือกอุปกรณ์"
2. เลือกอุปกรณ์ที่ต้องการวิเคราะห์
3. ระบบจะแสดงเมตริกของอุปกรณ์นั้น

**ตัวอย่างอุปกรณ์:**
- System-ECC800
- Cooling
- Multi-Func Sensor1
- Multi-Func Sensor2
- Integrated UPS Cabinet1

### 3.3 เลือกหมวดหมู่เมตริก 📂

- **ทั้งหมด:** แสดงเมตริกทุกหมวดหมู่
- **สิ่งแวดล้อม:** อุณหภูมิ, ความชื้น
- **ไฟฟ้า:** กระแส, แรงดัน, พลังงาน
- **ประสิทธิภาพ:** PUE, Load Factor

### 3.4 เลือกช่วงเวลา ⏰

- **1 ชั่วโมง:** ข้อมูลล่าสุด 1 ชั่วโมง
- **4 ชั่วโมง:** ข้อมูลล่าสุด 4 ชั่วโมง
- **24 ชั่วโมง:** ข้อมูลล่าสุด 1 วัน
- **3 วัน:** ข้อมูลล่าสุด 3 วัน
- **7 วัน:** ข้อมูลล่าสุด 1 สัปดาห์
- **30 วัน:** ข้อมูลล่าสุด 1 เดือน

## 4. การดูเมตริกตามหมวดหมู่

### 4.1 การแสดงผลหมวดหมู่

เมตริกจัดกลุ่มเป็น Accordion (พับเก็บได้) ตามหมวดหมู่:

- **🌡️ สิ่งแวดล้อม** (สีเขียว)
- **⚡ ไฟฟ้า** (สีส้ม)
- **📊 ประสิทธิภาพ** (สีฟ้า)
- **📈 อื่นๆ** (สีเทา)

### 4.2 การ์ดเมตริก

แต่ละเมตริกแสดงในรูปการ์ด ประกอบด้วย:

- **ไอคอน:** แทนประเภทของเมตริก
- **ชื่อเมตริก:** ชื่อเต็มของการวัด
- **หน่วย:** หน่วยการวัด (℃, A, kWh, %)
- **จำนวนข้อมูล:** จำนวน data points
- **วันที่อัปเดตล่าสุด:** เวลาข้อมูลล่าสุด

### 4.3 การโต้ตอบกับการ์ด

- **Hover:** วางเมาส์เหนือการ์ดเพื่อดูเอฟเฟกต์
- **คลิก:** คลิกการ์ดเพื่อดูรายละเอียด
- **สี:** สีของขอบการ์ดแสดงหมวดหมู่

## 5. การดูรายละเอียดเมตริก

### 5.1 การเปิด Dialog รายละเอียด

1. **คลิกที่การ์ดเมตริก** ที่ต้องการ
2. Dialog รายละเอียดจะเปิดขึ้น
3. แสดงข้อมูลครบถ้วนของเมตริกนั้น

### 5.2 ส่วนประกอบของ Dialog

#### การ์ดสถิติ (4 การ์ด)

1. **ค่าปัจจุบัน**
   - ค่าล่าสุดของเมตริก
   - ไอคอนแสดงแนวโน้ม (⬆️⬇️➡️)

2. **ค่าเฉลี่ย**
   - ค่าเฉลี่ยในช่วงเวลาที่เลือก
   - สีเขียวแสดงความเสถียร

3. **ค่าสูงสุด**
   - ค่าสูงสุดที่วัดได้
   - สีแดงแสดงจุดสูงสุด

4. **ค่าต่ำสุด**
   - ค่าต่ำสุดที่วัดได้
   - สีฟ้าแสดงจุดต่ำสุด

#### กราฟ Time-series

- **กราฟเส้น:** แสดงแนวโน้มตามเวลา
- **แกน X:** เวลา (อัตโนมัติปรับตามช่วงข้อมูล)
- **แกน Y:** ค่าเมตริกพร้อมหน่วย
- **จุดข้อมูล:** คลิกเพื่อดูค่าแต่ละจุด
- **Zoom:** ขยายกราหตามต้องการ

#### ปุ่มควบคุม

- **ปิด:** ปิด Dialog
- **ส่งออกข้อมูล:** ดาวน์โหลดข้อมูล (วางแผนไว้)

## 6. การเลือกช่วงเวลา

### 6.1 ช่วงเวลามาตรฐาน

| ช่วงเวลา | คำอธิบาย | การใช้งาน |
|---------|----------|-----------|
| 1 ชั่วโมง | ข้อมูลล่าสุด 1 ชั่วโมง | ติดตามเรียลไทม์ |
| 4 ชั่วโมง | ข้อมูลล่าสุด 4 ชั่วโมง | แนวโน้มระยะสั้น |
| 24 ชั่วโมง | ข้อมูลล่าสุด 1 วัน | วิเคราะห์รายวัน |
| 3 วัน | ข้อมูลล่าสุด 3 วัน | เปรียบเทียบแนวโน้ม |
| 7 วัน | ข้อมูลล่าสุด 1 สัปดาห์ | วิเคราะห์รายสัปดาห์ |
| 30 วัน | ข้อมูลล่าสุด 1 เดือน | วิเคราะห์รายเดือน |

### 6.2 การรวมข้อมูล (Auto-interval)

ระบบจะเลือกช่วงการรวมข้อมูลอัตโนมัติ:

- **≤ 6 ชั่วโมง:** รวมทุก 5 นาที
- **≤ 3 วัน:** รวมทุก 1 ชั่วโมง  
- **> 3 วัน:** รวมทุก 1 วัน

### 6.3 ตัวอย่างการใช้งาน

**สำหรับการติดตามปัจจุบัน:**
- เลือก "1 ชั่วโมง" หรือ "4 ชั่วโมง"
- เปิด Auto-refresh

**สำหรับการวิเคราะห์แนวโน้ม:**
- เลือก "7 วัน" หรือ "30 วัน"
- ดูกราฟในโหมดขยาย

## 7. การใช้งาน Auto-refresh

### 7.1 การเปิดใช้ Auto-refresh

1. **มองหาสวิตช์ "Auto Refresh"** ที่มุมบนขวา
2. **เปิดสวิตช์** (เปลี่ยนจากสีเทาเป็นสีฟ้า)
3. **ระบบจะอัปเดตทุก 30 วินาที**

### 7.2 การทำงานของ Auto-refresh

- **อัปเดตเมตริก:** อัปเดตข้อมูลเมตริกทุกหมวดหมู่
- **ไม่รบกวน Dialog:** ไม่ปิด Dialog ที่กำลังเปิดอยู่
- **แสดงสถานะ:** มีตัวบอกการโหลดขณะอัปเดต
- **ประหยัดแบตเตอรี่:** หยุดทำงานเมื่อแท็บไม่ active

### 7.3 เมื่อใช้ Auto-refresh

**ควรใช้เมื่อ:**
- ติดตามข้อมูลแบบเรียลไทม์
- ดูการเปลี่ยนแปลงอย่างต่อเนื่อง
- ใช้ช่วงเวลาสั้นๆ (1-4 ชั่วโมง)

**ไม่ควรใช้เมื่อ:**
- วิเคราะห์ข้อมูลประวัติศาสตร์
- ใช้อินเทอร์เน็ตช้า
- ต้องการประหยัดแบนด์วิธ

## 8. การแก้ปัญหาเบื้องต้น

### 8.1 ปัญหาการเข้าสู่ระบบ

**ปัญหา:** ไม่สามารถเข้าสู่ระบบได้

**การแก้ไข:**
1. ตรวจสอบ Username และ Password
2. ลองรีเฟรชหน้าเบราว์เซอร์ (F5)
3. ลบ Cache และ Cookies
4. ลองใช้เบราว์เซอร์อื่น

### 8.2 ปัญหาการโหลดข้อมูล

**ปัญหา:** ข้อมูลเมตริกไม่แสดง

**การแก้ไข:**
1. ตรวจสอบการเชื่อมต่อเครือข่าย
2. รีเฟรชหน้า (F5)
3. ลองเลือกไซต์และอุปกรณ์ใหม่
4. ตรวจสอบว่าเซิร์ฟเวอร์ทำงานปกติ

### 8.3 ปัญหากราฟไม่แสดง

**ปัญหา:** กราฟใน Dialog ไม่แสดงผล

**การแก้ไข:**
1. ปิดและเปิด Dialog ใหม่
2. ลองเปลี่ยนช่วงเวลา
3. ตรวจสอบว่ามีข้อมูลในช่วงเวลานั้น
4. รีโหลดหน้าเว็บ

### 8.4 ปัญหาประสิทธิภาพ

**ปัญหา:** ระบบช้า หรือค้าง

**การแก้ไข:**
1. ปิด Auto-refresh
2. เลือกช่วงเวลาสั้นลง
3. ปิดแท็บอื่นๆ ในเบราว์เซอร์
4. รีสตาร์ตเบราว์เซอร์

### 8.5 ปัญหาการแสดงผลบนมือถือ

**ปัญหา:** หน้าจอไม่ปรับตัวบนมือถือ

**การแก้ไข:**
1. หมุนหน้าจอเป็นแนวนอน
2. ซูมเอาท์ (pinch to zoom out)
3. ลองรีเฟรชหน้า
4. ใช้เบราว์เซอร์อัปเดตล่าสุด

## 9. เทคนิคการใช้งานขั้นสูง

### 9.1 การวิเคราะห์แนวโน้ม

1. **เลือกช่วงเวลายาว** (7-30 วัน)
2. **ดูกราฟในโหมดเต็มจอ**
3. **เปรียบเทียบหลายเมตริก**
4. **ใช้สถิติประกอบการวิเคราะห์**

### 9.2 การติดตามปัจจุบัน

1. **เลือกช่วงเวลาสั้น** (1-4 ชั่วโมง)
2. **เปิด Auto-refresh**
3. **ติดตามค่าปัจจุบันในการ์ดสถิติ**
4. **ตั้งแจ้งเตือนในเบราว์เซอร์** (ถ้าต้องการ)

### 9.3 การเปรียบเทียบไซต์

1. **เปิดแท็บใหม่** สำหรับไซต์ที่สอง
2. **เลือกไซต์ต่างกัน** ในแต่ละแท็บ
3. **เลือกเมตริกและช่วงเวลาเดียวกัน**
4. **เปรียบเทียบกราฟและสถิติ**

## 10. คำถามที่พบบ่อย (FAQ)

### Q1: ข้อมูลอัปเดตบ่อยแค่ไหน?
**A:** ข้อมูลในฐานข้อมูลอัปเดตทุกนาที แต่การแสดงผลใน UI อัปเดตตามการรีเฟรชหรือ Auto-refresh (30 วินาที)

### Q2: สามารถส่งออกข้อมูลได้หรือไม่?
**A:** ปัจจุบันกำลังพัฒนาฟีเจอร์นี้ คาดว่าจะมีใน Version ถัดไป

### Q3: ทำไมบางเมตริกไม่มีข้อมูล?
**A:** อาจเป็นเพราะ:
- อุปกรณ์ออฟไลน์หรือขัดข้อง
- ช่วงเวลาที่เลือกไม่มีการบันทึกข้อมูล
- การตั้งค่าการเก็บข้อมูลของอุปกรณ์

### Q4: กราฟแสดงผลผิดพลาดจะทำอย่างไร?
**A:** 
- ตรวจสอบหน่วยและช่วงค่า
- ลองเปลี่ยนช่วงเวลา
- ติดต่อผู้ดูแลระบบหากปัญหาเกิดซ้ำ

### Q5: ใช้งานบนมือถือได้หรือไม่?
**A:** ได้ ระบบออกแบบให้ Responsive แต่แนะนำให้ใช้หน้าจอขนาดใหญ่สำหรับประสบการณ์ที่ดีที่สุด
